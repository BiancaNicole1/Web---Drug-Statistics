1 Introduction

1.1 Purpose

RoDX is an interactive platform created by the two students mentioned earlier that provides a detailed perspective on drug consumption trends and data in Romania.
The purpose of this document is to present a detailed description of the functionalities as well as to specify the requirements of the web application. This app will give users the ability to analyze and visualize drug consumption data in Romania.

1.2 Document Conventions

We use bold text to define terms or highlight important concepts. This helps to draw the reader's attention to key points and make the document easier to understand. This document follows the software requirements documentation template according to the IEEE Software Requirements Specification.

1.3 Intended Audience and Reading Suggestions

This document is intended for students, teachers, developers or for any other user involved in the development, implementation, or use of the Drug Consumption Analysis Web Tool.

1.4 Product Scope

The Drug Consumption Analysis Web Tool aims to provide users with statistical insights into drug consumption trends in Romania over the past few years. It correlates drug consumption data with crime rates, medical emergencies, and prevention campaigns.

2. Description

2.1 Product Perspective

RoDX (Romanian Drug Explorer) is a web tool designed to analyze and visualize drug consumption data, created within the Web Technologies course. It interacts with external data sources for statistical information.

2.2 Product Functions

Each user will have access to the following functionalities:
- login (if the user has registered)
- register (if the user is new on the website)
- edit profile information
- view statistics (which leads to a page where you select a year and what data you want to be provided)
- download the generated charts as ".csv" or ".png"
- import/export/edit data from the database (only for admin users)
- an "About" Page (which leads to this documentation)
- a "Prevention" Page and one with information on some drugs
- a "Contact" Page

2.3 User Classes and Characteristics

Administrators: Manage system settings and access control; conduct in-depth analysis using the tool.
Logged Users: Access predefined reports and visualizations.
Unlogged Users: In order to acces other pages, they have to register first.

2.4 Operating Environment

The system is web-based and compatible with modern web browsers that carry HTML, CSS and JavaScript.

2.5 Design and Implementation Constraints

- Compliance with data protection regulations
- Compatibility with various data formats and sources

2.6 User Documentation

Comprehensive user documentation will be provided, including FAQs. Users can also acces this documentation for other information regarding the website.

2.7 Assumptions and Dependencies

- Availability of up-to-date and reliable data sources
- Internet connectivity for accessing external resources

The app needs an internet connection. The web communication standard is HTTP/HTTPS

4. System Features

Drug Consumption trend analysis Analysis and visualization of drug consumption trends in Romania over the past 3 years. Users will be able to explore and analyze data related to drug use patterns, changes over time, and demographic information.

4.1 Correlation analysis with crime rates

Allow users to analyze how drug consumption affects crime rates in Romania.

4.2 Correlation analysis with drug confiscations and medical emergencies

The web tool will enable users to explore the drug confiscations as well as medical emergencies related to drug use.

4.3 Generate charts and tables based on drug consumption trends

Registered users are able to view different types of charts and tables that are based on the information from the database which can be updated at any time by users that have the Administrator role.

4.4 Download the charts/tables generated

If an user is logged in, they can also download the generated charts and tables as ".csv" or ".png".
